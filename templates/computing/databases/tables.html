Designing our Table

Naming our Table

Although there are no formal rules for naming tables, it is helpful to have a convention.

Table names generally follow these conventions:

  • Names reflect the type of data held in the table. E.g. a table containing information on pupils, should be called PUPILS, not Stuff for people.
  • Names are usually plurals. E.g. CARS not CAR
  • Names are usually UPPER CASE. E.g. ORDERS not Orders

Chosing Fields

A table will contain the same fields for each record.

Each field will have a number of attributes:

  • Field Name - e.g. First Name, Date of Birth, etc
  • Data Type - e.g. CHAR, VARCHAR, INT, FLOAT, BOOL, etc
  • Length - the size allocated to store the field
  • Allow NULL - can the field be empty or a value is mandatory
  • PRrimary Key - this field will uniquely identify the record

Building Our Table

We can take our design, and convert it to a MySQL command to create the table in the server.

The command to create a table is:

CREATE TABLE TABLENAME ( field description )

For a full set of options that we can use to build tables, visit Here

The following code shows the SQL for the PUPILS table.


USE <you database name>;

CREATE TABLE PUPILS (
    pupil_id            INT NOT NULL,
    first_name          VARCHAR(255) NOT NULL,
    family_name         VARCHAR(255) NOT NULL,
    date_of_birth       DATE NOT NULL,
    address             VARCHAR(1024),
    distance_to_school  INT,
    gender              CHAR,
    absences            INT
);

Create the Pupils Table

Using the database admin page, and the code above, create the PUPILS table.
