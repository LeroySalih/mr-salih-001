Linking Tables.

Update

To be able to:

  • link 2 tables in a SELECT query.
  • identify primary and foreign keys.
In this module we are going to gain an understanding of how to Link data from multiple tables in a single query.

Task 1: Create the Execise DB

We are going to create a web page that lists all of the queries that you write.

  • Create a HTML repl page.
  • Using H1 tag, give the page a title, Linking Tables.
  • As you complete each task, copy the command to the index.html page with an appropriate heading.
  • You can use the "copy to clipboard" link to copy the SQL and results from the admin page.
  • Place the results inside a <PRE> tag to maintain spaces and line breaks.
  • Run the following script to create the database being used.
        use Nc1emhTgPx;

DROP TABLE PUPILS; 
DROP TABLE IF EXISTS HOUSE_POINTS;

CREATE TABLE PUPILS (
pupil_id	 INT		AUTO_INCREMENT PRIMARY KEY,
first_name   varchar(255) NOT NULL,
family_name  varchar(255) NOT NULL,
form         varchar(3) NOT NULL,
dob          date NOT NULL
);

CREATE TABLE HOUSE_POINTS (
record_id	INT        AUTO_INCREMENT PRIMARY KEY,
date_issued date       NOT NULL,
number_hp  INT         NOT NULL,
pupil_id    INT        NOT NULL
);

Linking Data

Example:

            
SELECT P.* 
FROM PUPILS as P;

SELECT HP.*, P.* 
FROM PUPILS as P, HOUSE_POINTS as HP
WHERE HP.pupil_id = P.pupil_id;
