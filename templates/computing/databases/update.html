Updating Data

To use our database, we are going to use the SQL language.

In SQL, to insert data, we use the UPDATE command.

This command has the following format.


              UPDATE <table_name> 
              SET <field_name>=<value>
              WHERE < criteria... >

Example:

for example, the following command will set the in_class field to TRUE for all records in the PUPILS table.

                
                  UPDATE PUPILS
                  SET in_class = TRUE

Example:

The following command will set the form field to '10C' for all records in the PUPILS table that currently have a field value or '9C'.

                
                  UPDATE PUPILS
                  SET form = '10C'
                  WHERE form = '9C'

PRIMARY KEYS

As you can see, it can become complex to uniquely identify a single row, based on the naturally occurring fields of a record.

Therefore, it is often necessary, to introduce a PRIMARY KEY. This is a field that is NOT NULL and UNIQUE within our table.

In order for a field to be a PRIMARY KEY, the current and future data must be guaranteed unique.

The following are usually considered unique:

  • IQAMA Numbers
  • Passport IDs
  • Numbers or Keys generated by the database

The following are usually NOT considered unique, and should not be used as PRIMARY KEYS:

  • Names
  • Phone Numbers
  • Date of Birth
