Inserting Data

To use our database, we are going to use the SQL language.

In SQL, to insert data, we use the INSERT command.

This command has the following format.


              INSERT INTO  
                (...)
              VALUES  
                ('Val1', 2, NULL)

for example...

              
                INSERT INTO PUPILS 
                  (first_name, family_name)
                VALUES 
                  ('Billy', 'Bob')

Whether this command works depends on how we have specified the table. If the table only has these fields, the data type is correct, then the command will work and the record will be added to the table.

Data Types

We know that every field will have a DATA TYPE, such as INTEGER, FLOAT, BOOL, VARCHAR, CHAR, etc. When we add data to a table, we must ensure that the data we are passing, matches the data type of the field.

  • INTEGERS and FLOATS are written as digits, e.g. VALUES (123, 12.4)
  • BOOLS are written as either words or digits, TRUE or 1 / FALSE or 0, with no quotes, e.g. VALUES (TRUE, FALSE, 1, 0)
  • CHARS and VARCHARs are written with either single or double quotes, e.g. VALUES('Mr', "Salih")
  • DATES are written is a number of different formats, but the easiest to use is as a string in the format YYYY-MM-DD, e.g. VALUES('2019-01-31')
